# 模式化学习与聚焦学习

## R — 原文 (Reading)

> "我觉得好的学习书，都是讲规律的。用《反向学习》的语言来说，它们讲的是模式。"
> "看家模式就是《反向学习》提出的一棵模式树的核心部分，也就是树干和大枝丫部分。看家模式和看家功夫一样，不会很多，也不需要很多，但是很管用。"
> "学习，也应该围绕人生愿景进行……要聚焦在擅长、热爱和机会的交叉之处。"
>
> — 刘澜，第2课、第9课、第117课

## I — 方法论骨架 (Interpretation)

本 skill 解决"学什么、学多少、聚焦哪里"三个问题，核心是六条相互咬合的判断：

1. **学习 = 模式 + 碎片**：学习的本质是用自己已有的少数模式去吸收新碎片，或在碎片中发现新模式。评判一种学习是否有效的试金石是——你在囤碎片，还是在用模式吸收碎片？博物馆式学习只有碎片无模式，是假学习。

2. **看家模式与必备模式**：看家模式是自己最擅长、反复使用、可用来吸收其他模式的核心模式（像看家功夫），数量少但管用。必备模式是"10点钟的模式"——简单、通用、能解决多数问题但不追求精准（如系统思考三层次）。学习策略是：聚焦少数必备 + 精通一两门看家。

3. **垒基学习（从 0 到 1 建立模式）**：进入一个新领域时先慢速建立初始模式，叫垒基学习。它难、慢、往往需要死记硬背，但重要——"如果说学习是用海绵吸水，垒基学习就是打造那块海绵"。没有"一"可举，谈何举一反三。

4. **聚焦三环（擅长 × 热爱 × 机会）**：绝大部分学习活动应围绕擅长、热爱、机会三环交叉地带进行。单纯聚焦擅长可能在造马车时代学造马车；单纯聚焦机会缺热爱撑不过至暗时刻；热爱是内在的，机会是外在的，擅长是连接内外的。三环交叉的专题要小（如"B2B 销售的目标管理"而非"目标管理"）。

5. **大盗式学习为主，小偷式为辅**：大盗式是围绕自己聚焦的问题读许多本书，提炼自己的答案，形成自己的模式树；小偷式是觉得有用就偷，不管体系。**做小偷的前提是当大盗**——大盗已有模式树，小偷只是偷树叶补充；纯做小偷只能搜集一堆碎片，无法变成大树。"读一本本好书"是误区，"读好一本书"才对。

6. **利用为主，探索为辅**：组织学习易犯"利用过度，探索不足"，但个人学习相反——大多数人犯的是"利用不足"。利用不足分两种：①不学习的人（利用不足+探索不足）；②买 100+ 门课的"天文学家"（利用不足+探索过度）。顿悟更可能发生在利用的领域。

## A1 — 书中的应用 (Past Application)

### 案例 1: 作者围绕"领导力"主题读 100+ 本书建立看家模式

- **问题**: 作者要建立自己在领导力上的看家模式，怎么读？
- **方法论的使用**: 大盗式学习——围绕"领导力"这个聚焦主题，慢速读前 5-10 本书（垒基学习，建立初始模式），然后围绕该主题认真读了不止 100 本书。读到第 11 本左右就开始快了，因为看家模式已建立，后续是用看家模式去吸收其他模式和碎片（小偷式补充）。
- **结论**: "围绕一个主题读 10 本书的过程，就是建立自己的看家模式的过程。如果建立起来了，读后面 90 本书就是用看家模式去吸收其他模式和碎片，阅读速度就快了。"
- **结果**: 作者现在读一本领导力的书，一般只要半小时到一小时——因为大多数内容已可被看家模式快速吸收，只剩一两点新东西值得"偷"。

### 案例 2: C 同学的博物馆式学习（反面诊断）

- **问题**: C 同学在群里分享过许多管理模型，看似学了很多，但提不出好问题，面对任何问题都能从自己的"博物馆"里搬出一个模型当答案。
- **方法论的使用**: 作者用"学习=模式+碎片"诊断——C 同学的学习是博物馆式学习，知识像博物馆展品，琳琅满目但彼此无联系。他学的内容太多，没有时间深究；想的是"怎么讲"而非"怎么用"，所以提不出好问题，像讲解员而非提问者。
- **结论**: C 同学缺的不是更多模型，而是少数能串联其他模式的看家模式。他需要先停下来，选一个专题，慢速垒基，建一棵模式树。
- **结果**: 作者借此案例提醒：学得多 ≠ 学得深；没有模式树的"博览"是低效的。

### 案例 3: K 同学研究"左宗棠"作为聚焦样本

- **问题**: K 同学想知道左宗棠的才干是否源于"纸上谈兵"，能否迁移到当今职场。
- **方法论的使用**: 作者建议用"聚焦一个样本"的学习方法——选一个具足够丰富性、蕴含相当数量重要模式的样本（如左宗棠），反复钻研。引叔本华："读完希罗多德就学完历史了，因为它已包括所有后来的历史所包括的东西"——样本不在于多，而在于是否把重要模式都呈现了出来。
- **结论**: 聚焦样本的目的不是浅层次学样本本身，而是发现样本背后的真正模式。需要读好几本左宗棠传记，甚至研究论文，而不是凭印象提一个"能不能"的问题。
- **结果**: 作者借此说明聚焦一个样本与"不学高手个人经验"不矛盾——前者是找模式，后者是照搬碎片。

### 案例 4: Y 同学读《漫画丰田生产方式》（反面）

- **问题**: Y 同学说自己读了《丰田生产方式》，但同学指出她读的是《漫画丰田生产方式》。
- **方法论的使用**: 作者用模式化学习标准判断——漫画和小视频一样，形式决定了它只能传递碎片，无法建构模式。真正的钻研需要围绕丰田生产方式读一些书（专家之作、论文），而非用漫画或"百度一下"替代。
- **结论**: 钻研方式过于轻松 = 没有真正钻研。模式化学习对媒介有要求——印刷文化才有助于模式化学习。
- **结果**: 作者借此提醒：媒介即讯息，用错媒介再勤奋也是假学习。

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. **面对一堆待学的东西不知道选哪个**：书单很长、课程很多、领域很广，需要判断该聚焦什么、不学什么。
2. **学了很多却建不起自己的体系**：感觉知识散乱、彼此无联系，怀疑自己在做博物馆式学习。
3. **想围绕一个主题深读，但不知道读到什么程度算够**：例如"我想研究销售/管理/教育孩子，该读多少书？怎么读？"
4. **同时学多门功夫/多个领域，效果都不好**：怀疑自己分散了精力。
5. **想找到自己的看家模式**：感觉没有一个能反复使用、吸收其他模式的核心工具。
6. **判断要不要换学习方向/换老师**：现在学的东西是否还在三环交叉地带？是否该换地图？

### 语言信号（用户的话里出现这些就应激活）

- "我该学什么" / "学多少才够" / "同时学几样合适"
- "学了很多但用不起来" / "感觉知识很散" / "建不起体系"
- "想围绕一个主题深读" / "想找到自己的看家模式" / "我的模式树是什么"
- "这本书值不值得读透" / "该不该换老师" / "要不要换方向"
- "what should I learn" / "how many books" / "pattern library" / "focus topic"
- "博物馆式学习" / "大盗式学习" / "聚焦三环" / "看家模式"

### 与相邻 skill 的区分

- 与 `si-wen-xuexi-fa`（四问学习法）的区别：那个是"听→找→变→用"四步整体流程，解决"学到的东西怎么走完四步"；本 skill 解决"该学什么、学多少、聚焦哪里"，是上游的选材决策。二者互补——本 skill 决定学什么，四问学习法决定怎么吸收。
- 与 `faxian-moshi`（发现模式）的区别：那个是单次找模式的具体动作（四问学习法第二步"找"的深化），解决"这次怎么找出背后的模式"；本 skill 是长期模式库建设——选什么领域、读多少本书、建什么看家模式，是战略性而非战术性的。

## E — 可执行步骤 (Execution)

当 skill 被激活后，agent 应引导用户按以下步骤走：

1. **盘点已有模式，识别看家模式**
   - 让用户列出自己反复使用、最擅长、能用来吸收其他知识的 1-3 个模式（如四问学习法、PDCA、某种业务模型）。
   - 完成标准：能说出每个看家模式"我用它吸收过哪些碎片"，且能区分"我知道这个模式"和"我用它吸收过碎片"——后者才是真看家。
   - 判停条件：若用户列不出任何看家模式，提示他可能还没完成垒基学习，应先在一个领域慢速读 5-10 本书建立初始模式，再谈聚焦。

2. **按聚焦三环选学习领域与专题**
   - 引导用户分别列出：我擅长什么（具体到 B2B 销售、文案写作，而非"沟通"）/ 我热爱什么（具体到金钱、自主感、育人之乐，而非"成功"）/ 机会在哪里（行业性/职能性/角色性）。找三环交叉的小专题（如"B2B 销售的目标管理"而非"目标管理"）。
   - 完成标准：产出一个具体到可操作、三环交叉的专题，且能说明为什么是三环交叉而非单环。
   - 判停条件：若专题太大（如"如何管理"）或脱离自己实际工作生活，回到三环重新选。

3. **围绕聚焦专题做大盗式学习**
   - 引导用户：①先慢速读 5-10 本书建立模式（垒基学习），②再围绕该专题读够 10-100 本书，③边读边输出（讲课/写作），建构自己的模式树。判断是否已建立看家模式的标准是——读第 11 本书时是否变快了。
   - 完成标准：选定一个聚焦专题 + 列出已读/待读的 5-10 本核心书 + 一次输出（讲一课/写一篇）。
   - 判停条件：若用户只想"快速读书"而不愿慢速垒基，提示他前 10 本必须慢，否则建不起看家模式。

4. **警惕博物馆式学习，定期自检**
   - 让用户自问：我学的东西彼此有联系吗？我学的是为了"用"还是为了"讲"？我面对任何问题是不是都能从博物馆搬出一个模型当答案？
   - 完成标准：能说出最近学的 3 个东西如何挂在同一棵模式树上，而不是孤立展品。
   - 判停条件：若用户只能罗列学过的模型而说不出它们的关系，提示已陷入博物馆式学习，停止学新东西，先整合已有的。

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- **单次内容的吸收流程**：用户面对一篇文章/一节课想"怎么用起来"，用 `si-wen-xuexi-fa`，不是本 skill。
- **单次找模式动作**：用户卡在"这个背后是什么模式"，用 `faxian-moshi`，不是本 skill。
- **纯信息查询**：用户只想知道某个事实，不需要选材决策。
- **没有"听"够就想"找"模式**：脑子里的模式储备太少（垒基学习没做），强行聚焦也会变成空想——此时应先去多读几本该领域的基础书。

### 作者在书中警告的失败模式

- **博物馆式学习（ce01）**：学了许多模型但彼此无联系、无深度，只为"讲给别人听"或"看起来懂得多"。面对任何问题都能从博物馆搬出一个模型当答案，但不深究、不真用。信号：学的内容彼此无联系、想的是"怎么讲"而非"怎么用"、提不出好问题。
- **同时学太多本领（ce09）**：没有建立看家模式前就分散精力，同时练少林和武当，买 100+ 门课。结果要么什么都没学到，要么学成浅层次谈论各种知识的"天文学家"。信号：没有看家模式、探索过度利用不足、买的课都没学完。
- **读一本本好书（ce10）**：追求读"好书"（评分高、高手推荐）的数量，却没有一本读透、读成看家模式。把"好书"当客观标准，忽略学习力第一公式（好书因人而异）。信号：追求读书数量、选书只看评分、没有一本书反复读。
- **单纯小偷式学习（ce27）**：到处偷碎片，没有自己的模式树。判断不出什么对自己真正有用，搜集的只是一堆树叶和小枝丫，没法变成大树。信号：到处找"有用"的碎片、学了很多但无法整合。
- **读讲课型老师而非专家（ce26）**：跟着"讲课型"老师学，学到的是浅知识（像记者/编辑讲给外行听），以为这就是领域精华。要成为专家必须读专家之作。信号：学到的知识不够深刻、没读过该领域专家之作、不知道领域顶尖思考是什么。
- **用漫画/百度"钻研"（ce21）**：用漫画替代原著、用百度替代钻研。媒介形式决定它只能传递碎片，无法建构模式。信号：钻研方式过于轻松、觉得"知道概念"就等于"掌握概念"。

### 作者的盲点 / 时代局限

- **强依赖前作**：本书是《反向学习》的实战篇，"模式""看家模式""模式树"等核心术语在前作已建立，独立读者可能不熟悉——使用本 skill 时若用户不熟悉这些词，先用日常语言解释。
- **案例样本窄**：约 50 个学员案例多为企业管理者/创业者/职场人，对学生、蓝领、自由职业者、退休人群的适用性未充分验证。三环（擅长×热爱×机会）对还在探索阶段的年轻人可能难以直接套用。
- **"模式"普适性假设**：作者假设所有有效学习都能归结为"找模式"，但创造性突破、灵感、默会知识的获得未必都能被模式化（如艺术、诗、直觉）。强行找模式可能把本无模式的偶然现象套上模式，产生虚假规律感。
- **"用"的优先性假设**：作者整体导向工具主义，把"用"置于"知"之上，可能贬低纯粹求知价值——对为知识而知识的场景（哲学思辨、审美、纯数学）应放宽"用"的标准。
- **个人权威色彩**：部分判断（"我就是这么认为的"）依赖作者个人权威而非论证，读者应保持参考答案思维方式。

### 容易混淆的邻近方法论

- **费曼学习法**：强调"教别人"来检验理解，相当于本 skill 中"通过输出学习"的一种形式（讲课），但不覆盖"学什么、学多少、聚焦哪里"的战略决策。
- **主题阅读法（Syntopical Reading，出自《如何阅读一本书》）**：与本 skill 的"大盗式学习"形似——都是围绕一个主题读多本书。区别在于：主题阅读法强调比较不同作者的观点得出自己的结论，本 skill 强调建构一棵自己的模式树，且强调"读好一本书"（建看家模式）优先于"读多本书"。
- **刻意练习**：强调在舒适区之外的高难度练习+反馈，是"怎么练"的方法论；本 skill 是"学什么"的方法论。二者可配合——选定聚焦专题后，用刻意练习的方式练看家模式。
