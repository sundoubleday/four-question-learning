# 发现模式（分析问题的核心步骤）


## 刘澜视角（作者思维透镜）

> **体系定位**：本 skill 是"五项修炼"第四项——模式化学习的方法落地（专家握模式，新手只有碎片）。
> **作者镜片**：模式化镜片——模式如磁铁，碎片如铁屑；"我知道"若是模式知识，就能吸住"我不知道"的碎片。
> **招牌比喻**：磁铁吸铁屑——零散经验是铁屑，背后的共性/本质才是磁铁。
> **边界提醒**：建"模式树"有门槛，对新手是慢思考的硬仗，别指望一次读懂就成专家。

## R — 原文 (Reading)

> "分析问题，不仅第一步不是探究原因，第二步也不是探究原因。第二步是发现模式……第二步要问这样的问题：你所说的问题背后的模式是什么？第二步实际上是在找原因。但不是直接找原因，而是在模式的指导下找原因。第二步也是在找答案，在模式的指导下找答案。发现模式，既是找原因，也是找答案。"
>
> — 刘澜，第19-22课

> "如何找出更好的参考答案呢？从三个方向找。第一个方向是从原文的逻辑中找……第二个方向是从脑子中已有的模式里找……第三个方向是从自己想要解决的问题中找。"
>
> — 刘澜，第95课

## I — 方法论骨架 (Interpretation)

"发现模式"是四问学习法第二步"找"的展开，也是分析问题四步（界定问题→**发现模式**→提出方案→执行方案）的核心。

关键洞察：**分析问题的第二步不是直接找原因，而是先发现模式。** 直接找原因容易陷入就事论事的碎片；先找到模式，再在模式指导下找原因和找答案，才能挖到可迁移的本质。发现模式同时承担两个功能——既是找原因（解释为什么会这样），也是找答案（指导该怎么做）。

模式藏在哪里？藏在碎片表面之下，要"找"不是"想"。"想"是任何念头都算，"找"要求有目标地翻开表面。从三个方向找：
1. **从原文/经验的逻辑中找**——把显性要点及要点之间的关系都纳入，别漏掉关系。
2. **从脑中已有模式中找**——不管原文讲了什么，问自己"关于这类事我知道哪些模式"，可用"向上归类"（如把"政治工作"归为"激励"）。
3. **从要解决的问题中找**——把"听到的"和"要用的"并列，问"如果它们是同一回事，共同模式是什么"。

三个方向互补，没有标准答案，但有更优的参考答案。判断找到的模式好不好，用两个标准：能否帮理解"一类"事物的本质？能否指导"一个"情境中的行动？

## A1 — 书中的应用 (Past Application)

### 案例 1: P 同学找"解放军政治工作"背后的模式
- **问题**: P 同学读到"政治工作是我军生命线，两个原则是思想教育和模范带头"，要找到可迁移的模式。
- **方法论的使用**: 他"找"到的是作者的领导力口诀"我来"。但作者指出这不够——"我来"忽略了原文两个原则中的一个，且没抓住"要点之间的关系"。
- **结论**: 改用"五教"模型（说教/身教/管教/请教/传教）更深：说教和身教正好对应原文两个原则，还能延伸出"另外三个教该不该有"的进一步思考。
- **结果**: 示范了"从原文逻辑找"（要把两个原则及其关系都纳入）和"从已有模式找"（五教模型比"我来"更结构化）的区别——后者更优。

### 案例 2: "摇尾巴"不是模式
- **问题**: 小狗写书总结"摇尾巴就能讨人喜欢"，马能照搬吗？人能照搬吗？
- **方法论的使用**: 作者追问"摇尾巴背后有没有本质模式？这个模式是没有尾巴可摇的我也能学的？"——这就是"从要解决的问题找模式"。
- **结论**: 直接照搬表面行为（摇尾巴）是照搬碎片；要找到背后模式（如"主动示好""建立情感联结"）才能跨物种迁移。
- **结果**: 海底捞的店长照搬摇尾巴会失败，因为没找到模式。这是"发现模式"的反面教材——照搬碎片。

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. 分析一个问题时，列了一堆原因却理不出头绪，找不到根本原因。
2. 看到别人的成功经验/做法，想学但直接照搬又怕不适合自己。
3. 读完一段内容，能复述但说不出"这背后的规律是什么"。
4. 同一个问题反复出现，想找到底层模式一劳永逸。
5. 想把一个领域的经验迁移到另一个领域，但不知道迁移什么。

### 语言信号 (用户的话里出现这些就应激活)

- "这背后的模式/规律/本质是什么"
- "为什么总是这样" / "根本原因是什么"
- "我照搬了XX的做法但没用"
- "这个经验能不能用到我这儿"
- "怎么迁移" / "有什么共性"
- "underlying pattern" / "what's the model" / "transferable"

### 与相邻 skill 的区分

- 与 `si-wen-xuexi-fa`（四问学习法）的区别：那个是"听→找→变→用"完整流程；本 skill 专门展开第二步"找"——当用户已经"听"到了内容、卡在"找不到模式"时，从主框架切换到本 skill 深化。
- 与 `moshi-jujiao-xuexi`（模式化与聚焦）的区别：那个解决"该积累哪些模式、学多少"（模式库建设）；本 skill 解决"此刻面对这个碎片，怎么挖出模式"（单次提取动作）。
- 与 `cankao-daan-siwei`（参考答案思维方式）的区别：那个是思维底座（不把任何答案当标准答案）；本 skill 是找模式的具体操作，产出的模式本身就是"参考答案"——二者是"产出什么"和"怎么看待产出"的关系。

## E — 可执行步骤 (Execution)

当 skill 被激活后，agent 应引导用户按以下步骤找模式：

1. **明确"要找什么"——先界定问题/碎片**
   - 让用户先说清：面对的是什么现象/经验/内容？要解决什么问题？
   - 完成标准：能一句话说清"我在找什么的背后模式"。
   - 🔴 检查点：用户说不清"在找什么的背后模式"之前，不进入方向展开。若用户连"听到了什么"都模糊，先回到 `si-wen-xuexi-fa` 第一步"听"补素材，不要硬找。

2. **从三个方向找模式**
   - 顺序：先方向一（原文逻辑）打底，确认要点及其关系；再方向二（已有模式）和方向三（要解决的问题）并行展开。若用户已有明确模式储备，可先从方向二切入。
   - **方向一·原文逻辑**：把显性要点及要点**之间的关系**都列出。问"这些要点之间是什么关系？配合？对立？递进？"——关系往往是模式所在。
   - **方向二·已有模式**：不管原文，问自己"关于这类事，我脑子里有哪些模式？"若无，用"向上归类"（把具体问题归到更大类别，如"政治工作"→"激励"）再找。
   - **方向三·要解决的问题**：把"听到的"和"要用的"并列，问"如果它们是同一回事，共同模式是什么？"
   - 完成标准：至少从一个方向找到一个候选模式，且能说清它比表面内容深在哪。
   - 判停条件：若三个方向都挖不出任何候选模式，停止硬找。改做"向上归类"练习或回 `moshi-jujiao-xuexi` 补垒基学习，不要产出假模式凑数。

3. **用好模式两标准检验**
   - 标准1（理解"一类"）：这个模式能帮理解一类事物的本质吗？还是只解释了这一个案例？
   - 标准2（指导"一个"）：这个模式能指导我在这个具体情境中的行动吗？还是太抽象用不上？
   - 完成标准：两个标准都基本满足。
   - 🔴 检查点：找到 1 个候选就停是不达标的。至少产出 2-3 个候选再选最优，并提醒"还有没有更优的参考答案"——找模式没有标准答案，但有更优解。

4. **产出参考答案**
   - 把找到的模式写成一个可表述的结构（一句话+几点要素），明确它是"参考答案"而非标准答案。
   - 完成标准：产出一个明确的模式表述，可交给 `si-wen-xuexi-fa` 的第三步"变"继续。
   - 判停条件：若用户把模式写成"要努力""要复盘"这类无法指导行动的空话，退回标准2重检，要求落到可操作的具体要素。

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- **问题已清楚只需执行**：已经知道模式也知道怎么做，只差动手——直接执行，不必再"找"。
- **纯事实核对**：查一个数据、确认一个信息，不需要找模式。
- **情绪问题**：用户在情绪中，需要共情而非分析模式。
- **垒基学习阶段（软边界）**：用户脑中模式储备太少（还没建立看家模式），强行"找"会变空想。但若用户已有明确的"找"意图（如"看什么都觉得是模式但说不清"），**可激活本 skill**——先用方向二（已有模式）做简单练习建立初始储备，同时提示需补垒基学习（见 `moshi-jujiao-xuexi`）。不要直接拒绝。
- **没有任何碎片可找**：用户连"听到了什么"都说不清，先回到四问学习法第一步"听"。

### 作者在书中警告的失败模式

- **照搬摇尾巴**（ce05）：直接照搬成功者表面做法，没找背后的模式。看到"那么做"成功就模仿"那么做"，忽略幸存者偏差和情境差异。马学狗摇尾巴会失败。
- **第二步"想"得太随意**（ce13）：把"找"退化成"想"，任何念头都算"想"，没翻开表面。P 同学找到"我来"就停下，没继续找更深的"五教"。
- **找到一个就停**：找模式没有标准答案但有更优解，找到一个参考答案就停下来，会错过更好的模式。
- **只找不"变"不"用"**：找到模式就以为完成了，不做情境调整和落地——那是四问学习法后两步的事。
- **从原文逻辑找时漏掉"关系"**：只列要点不列要点之间的关系，而关系往往是模式所在（如政治工作两个原则的"配合"本身就是模式）。

### 作者的盲点 / 时代局限

- 作者假设所有有效经验都能找到模式，但创造性突破、灵感、默会知识未必都能模式化——对艺术、直觉类内容应承认"可能没有可迁移模式"。
- 强行找模式可能把本无模式的偶然现象套上模式，产生虚假规律感——找到的模式要用"好模式两标准"和实际"用"来检验。
- "向上归类"技巧依赖用户已有的模式储备，对初学者可能无效——此时应先做垒基学习。

### 容易混淆的邻近方法论

- **第一性原理**：从基本事实向上推演，与本 skill"从原文逻辑找"有重叠，但第一性原理强调"回到不可再分的基本真理"，本 skill 强调"翻开表面找藏着的关系结构"。
- **5Why**：连续问"为什么"找根本原因，是"直接找原因"的代表——作者明确批评这种做法，主张先找模式再在模式指导下找原因。
