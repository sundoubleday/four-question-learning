# 四问学习法（听→找→变→用）

> 📌 **整套 skill 的使用地图**：[`assets/usage-map.png`](assets/usage-map.png) —— 以四问学习法为主轴，"听/找/变/用"四步各挂一个深化工具，底部为排障与升级工具，一图看清 8 个 skill 的关系与调用入口。

## R — 原文 (Reading)

> "《反向学习》最为强调的四问学习法可以概括为'听想变用'，落脚点就是用。用不起来的学习，往往是假学习。"
> "我前几天突然觉得可能找到了一个比较好的解决方案，用'找'字来替代'想'字……我们脑子里浮现出任何一个念头，都可以说是'想'，但'找'不是。"
>
> — 刘澜，《学会如何学习》第8课、第94课

> "我为深层迁移专门发明了一个工具——四问学习法。四问学习法由四个问题组成：我听到什么？我想到什么？我变成什么？我用到哪里？"
> "学用脱节表现为两个方面：要么学而不用，要么学而乱用，前者的危害更严重。为了替代它，我们要进行深层迁移（第五项修炼）。这是模式化学习的深化，即把模式知识应用于新的情境之中。"
>
> — 刘澜，《反向学习：学习力的五项修炼》第五章"深层迁移"（机械工业出版社，2025）

## I — 方法论骨架 (Interpretation)

四问学习法是把任何"碎片"（书本、经验、他人观点、生活事件）转化为"自己能用得上的东西"的主轴流程。在《反向学习》的五项修炼体系中，它是**第五项修炼"深层迁移"的专属工具**——深层迁移要解决"学用脱节"（要么学而不用、要么学而乱用）这个坏习惯，把模式知识应用到新情境里；四问学习法就是执行这个迁移的四步流程。分四步：

1. **听**——我听到/看到什么？如实接收对方的显性内容，不急于评判。
2. **找**（作者把"想"修正为"找"）——这背后藏着什么模式？"想"太随意，任何念头都算"想"；"找"要求有目标地翻开表面，把藏着的结构挖出来。这是四步里最难、最关键的一步，有专门的深化 skill `faxian-moshi`。
3. **变**——这个模式怎么根据我的情境调整？模式不能生搬硬套，要"变"成适合我的版本。
4. **用**——我用到哪里？具体到下一步行动。"用"既是试金石（用不起来的学习是假学习），也是脚手架（思考怎么用，反而把知识学得更深）。

四步顺序不可变：没有"听"够就"找"是空想（思而不学则殆）；只"听"不"找"是把碎片当知识；"找"到模式不"变"是生搬硬套；不"用"则前三步全废。越往后越难、越要靠自己。

## A1 — 书中的应用 (Past Application)

### 案例 1: P 同学读"解放军为什么能赢"
- **问题**: P 同学读到"政治工作是我军的生命线"，两个原则是思想教育和模范带头，想用到自己的项目管理中。
- **方法论的使用**: 他按四问学习法写：听到（两个原则）→想到（作者的领导力口诀"我来"）→变成（企业运营既要思想教育又要表率）→用到（项目管理）。
- **结论**: 方向正确，但作者指出第二步"想"得太随意——"我来"不够模式化，没有挖到藏着的东西。若改用"五教"模型（说教/身教/管教/请教/传教）会更深。
- **结果**: 作者借此提出把"想"改为"找"，并衍生出"找模式的三个方向"（见 `faxian-moshi`）。

### 案例 2: 作者"变着花样"用四问学习法分析学员案例
- **问题**: 全书约 50 个学员案例，每个都要分析"学得对不对"。
- **方法论的使用**: 作者反复用四问学习法作为分析工具——学员"听"到了什么、"找"到什么模式、"变"得是否切己、"用"得是否具体。
- **结论**: 四问学习法不仅是学习工具，也是诊断工具：哪一步出问题，学习就在哪一步失效。
- **结果**: 示范了同一个工具在不同情境下的灵活使用，是"看家模式"的范例。

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. 读完一本书/一篇文章/听完一节课后，想真的用起来而不是读完就忘。
2. 听到别人分享一个观点或经验，想吸收成自己的。
3. 学了一堆却"用不起来"，怀疑自己在假学习。
4. 面对一个碎片信息（金句、案例、新闻），想把它变成能指导行动的东西。
5. 想判断自己或他人的学习卡在哪一步。

### 语言信号 (用户的话里出现这些就应激活)

- "这本书我读完了，但不知道怎么用" / "学了很多用不上"
- "怎么把学到的东西用起来" / "学以致用"
- "听到一个观点，我该怎么吸收"
- "这个经验对我有什么启发"
- "how do I apply this" / "put into practice" / "absorb"

### 与相邻 skill 的区分

- 与 `faxian-moshi`（发现模式）的区别：本 skill 是四步整体流程；`faxian-moshi` 专门深化第二步"找"，当用户卡在"找不到背后的模式"时切换到那个 skill。
- 与 `moshi-jujiao-xuexi`（模式化与聚焦）的区别：那个 skill 解决"该学什么、学多少"，本 skill 解决"学到的东西怎么走完四步"。
- 与 `xuexi-tiwen`（学习要有体温）的区别：那个 skill 检验学习是否真发生（认知/情感/行为三维度），本 skill 是流程工具；二者配合——四问学习法是骨架，体温是血肉。

## E — 可执行步骤 (Execution)

当 skill 被激活后，agent 应引导用户按以下步骤走：

🔴 **CHECKPOINT · 起点确认**：激活后先确认用户手上的"碎片"是什么形态——一本书/一节课/一个经验/一条金句/一条新闻？并确认它已"听"到什么程度（读完了？只听了片段？）。若用户还没"听"够（只读了一句话、只听了个标题），先提示去多"听"再回来，不要直接跳到第二步"找"——否则会变成"思而不学"（ce18）。确认后进入第 1 步。

1. **听——如实接收**
   - 让用户用一两句话复述"听到/看到"的显性内容（对方说了什么、书里写了什么）。
   - 完成标准：能区分"对方明确说的"和"我自己加的判断"。
   - 判停条件：若用户复述时夹带大量自己的推断、分不清"对方说的"和"自己加的"，提示这是"学而不思"的反面——听成了自己的预设，回到 🔴 起点确认，请其只说对方原话。

2. **找——挖背后的模式** ★ 最关键
   - 引导用户问："这背后藏着的模式是什么？"强调是"找"不是"想"——要翻开表面，不是随便冒念头。
   - 可调用 `faxian-moshi` skill 的三个方向（原文逻辑/已有模式/要解决的问题）辅助。
   - 完成标准：找到至少一个比表面内容更深的"结构"，且能说出它为什么是模式而非碎片。
   - 判停条件：若用户只能复述表面内容、找不到任何深层结构，提示这可能是"思而不学"或内容本身无模式，回到第一步多"听"几个来源。

3. **变——根据情境调整**
   - 问："这个模式怎么根据你的情境变形成你能用的版本？"提示不要生搬硬套。
   - 完成标准：产出一个针对用户具体情境的、调整过的模式版本，而非原文照搬。
   - 判停条件：若用户只是把原文模式换了个主语复述（"我要用五教模型"但没说怎么结合自己的具体情境），提示这是生搬硬套（不"变"），回到第二步检查找到的是不是太浅——可能只挖到了表面，需要调用 `faxian-moshi` 再深挖一层。

4. **用——落到下一步行动**
   - 问："你具体用到哪里？下一步做什么？"要求具体到可执行的动作，不接受"我会注意"这类模糊承诺。
   - 完成标准：有一个明确的、近期可执行的行动；能说清"用了之后怎么检验是否有效"。
   - 判停条件：若用户说不出具体行动，往往说明前三步没走完，回到第二步重新"找"。

## B — 边界 (Boundary) ★

### 不要在以下情况使用此 skill

- **纯信息查询**：用户只是想知道某个事实（"这本书作者是谁"），不需要四步流程。
- **单纯记忆背诵**：背单词、记公式等机械记忆任务，四问学习法是负担。
- **情绪倾诉/安慰**：用户在情绪中，需要的是共情而非流程化分析。
- **没有"听"够就想"找"**：用户脑子里的模式储备太少（垒基学习没做），强行"找"会变成空想——此时应先去多"听"（读书、跟老师），见 `moshi-jujiao-xuexi` 的垒基学习。

### 作者在书中警告的失败模式

- **思而不学则殆**（ce18）：没"听"够就"找"，思没有学的根基，思也不深刻。四步顺序不能变。
- **第二步"想"得太随意**（ce13）：把"想"当自由联想，任何念头都算"想"。作者因此把"想"改为"找"——"找"要求有目标、翻开表面。
- **用得太宽泛**：第四步"用"如果只说"用到工作中"而拿不出具体动作，等于没用。P 同学的"用到项目管理"就是用得太宽泛。
- **学而不思**（与思而不学相反）：只"听"不"找"，把碎片当知识，停留在被动学习。
- **不做习题**（ce28）：读书但不做习题=没有"想"和"用"的环节，是假学习。

### 作者的盲点 / 时代局限

- 作者强依赖前作《反向学习》的概念体系，独立读者可能不熟悉"模式""看家模式"等术语——使用本 skill 时若用户不熟悉这些词，先用日常语言解释。
- "用不起来的学习一定是假学习"把"用"置于"知"之上，可能贬低纯粹求知价值——对为知识而知识的场景（如哲学思辨、审美）应放宽"用"的标准。
- 案例多为企业管理者/创业者，对学生、蓝领的适用性需用户自行迁移。

### 容易混淆的邻近方法论

- **PDCA/PDSA**（戴明环）：与本 skill 的"听想变用"形似，但 PDCA 是质量管理循环，本 skill 是学习循环；作者指出四问学习法其实就是 PDSA 中 P（plan）的形成过程。
- **费曼学习法**：强调"教别人"来检验理解，相当于本 skill 的"用"的一种形式，但不覆盖完整四步。
